GitHub Pages Setup

Last updated: 2026.05.26

This is a reference sheet for how I (1) set up my blog using GitHub Pages and (2) editing a HTML/CSS website across multiple devices using Git and the terminal.

Part 1. SSH Setup (Authentication)

GitHub does not allow username/password authentication for Git operations anymore. Instead, it uses either (1) personal access tokens or (2) SSH keys. This tutorial uses SSH.

Step 1.1. Generate SSH key

ssh-keygen -t ed25519 -C "your_email@example.com"

Press Enter for default options unless you want to customize the file location.

Step 1.2. Start SSH agent

eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

This starts the SSH agent and loads your private key so Git can use it automatically.

Step 1.3. Add SSH key to GitHub

Copy your public key:

cat ~/.ssh/id_ed25519.pub

Then go to:

GitHub → Settings → SSH and GPG keys → New SSH key

Paste the key and save.

Step 1.4. Test connection

ssh -T git@github.com

If successful, you will see:

Hi USERNAME! You've successfully authenticated, but GitHub does not provide shell access.

This is expected. It only confirms authentication, and also a reminder that GitHub is not a shell server.

Part 2. GitHub Repository and Pages Setup

Step 2.1. Create a new repository

Go to GitHub and create a new repository:

  • Name: anything (e.g. my-website)
  • Visibility: Public (required for GitHub Pages)
  • Do NOT initialize with README (optional but cleaner for terminal workflow)

Step 2.2. Connect local project to GitHub

Go to your website folder.

cd /path/to/your/website

Once you are in your website's folder, initialize git and connect it to your repository.

git init
git remote add origin git@github.com:USERNAME/REPO.git
git branch -M main

Step 2.3. Upload your website

git add .
git commit -m "Initial website upload"
git push -u origin main

At this point, your code is stored on GitHub.

Step 2.4. Enable Pages

Go to your repository:

Settings → Pages

Then set:

  • Source: Deploy from branch
  • Branch: main
  • Folder: / (root)

Save the settings.

Step 2.5. Access your website

Your site will be live at:

https://USERNAME.github.io/REPO/

Part 3. Website Editing Workflow

Always sync the website folder before making any edits to the website. This is especially true if you edit the website through multiple devices.

Step 3.1. Sync latest version

git pull

Step 3.2. Make edits

Edit HTML, CSS, images, etc.

Step 3.3. Save and publish changes

git add .
git commit -m "Describe update"
git push
